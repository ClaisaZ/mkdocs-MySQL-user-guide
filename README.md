# Purpose of this doc

This doc walks users through using [MySQL Workbench](https://www.mysql.com/products/workbench/) at a basic level on Windows. It explains how to complete common beginner tasks with Workbench, including:

- how to create a connection
- how to create a schema (database)
- how to create a table
- how to insert data
- how to run a query
- how to edit data

**Important**: This is not a guide on how to write MySQL to manage databases. It is a guide on how to use MySQL Workbench’s GUI to complete basic database tasks and does not go in depth.

## Intended reader

We wrote this for beginner users, especially students who are new to SQL tools and anybody who hasn't used a database GUI before. It's designed for reader who want clear and simple instructions with visuals and explanations.

## What this doc includes

This doc project includes:

- an introduction section
- separate instruction pages for basic Workbench tasks
- a glossary of important terms
- a troubleshooting guide for common issues
- screenshots for visual help throughout the guide

## How we planned and collaborated

When we got the assignment first, we decided what tasks in Workbench a beginner user would probably do, this includes creating a connection, schema, table, and inserting data, running queries and editing data.

We split the work to different pages and used Github to manage updates to the project. We tried to avoid conflicts too, so we worked on separate files/sections, although we still ended up with some conflicts by accident. We also reviewed each others work before pushing.

## Methods and process

We created this guide using Markdown, Material for MkDocs, and custom CSS/JS. We made the pages in a way to follow instruction writing principles for the web, including:

- clear headings and subheadings
- numbered steps with one action per step
- notes and warnings where needed
- screenshots to support each task
- concise explanations for beginner users

We also organized the pages so users can complete each task in order.

## Using MkDocs

We used Material for MkDocs because it gives clear navigation and support for admonitions which we used. And overall it has a very clean layout for web docs. We also customized the site with CSS so the pages would be easier to read, and because it looks better.

## Using Markdown

We wrote this in Markdown because it made it easier to structure headings, steps, notes, images, and code examples with a consistent format and it looks clean.

## How we improved readability

To make the steps easier to follow we put screenshots throughout the pages so users can visually understand the instructions instead of just reading the text, it makes it easier to follow along. We also wrote simple and direct so that way each step is easier to understand.

## Tone

We're keeping this friendly, simple, and practical, like walking a classmate through setup. 👍

## Conclusion

We had to think a lot about how beginners learn a new software tool, and what kind of support they need in a guide to using something for the first time, it made us think very carefully about that. It also helped us practice writing user documentation, it's a good skill for us in the future.
